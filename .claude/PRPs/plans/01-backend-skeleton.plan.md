# Plan 01: Backend Skeleton & Persistence

## Summary
Stand up the VibeMaestro local backend foundation: Bun + Turborepo monorepo, Electron app shell (electron-vite), tRPC bridged over Electron IPC, Drizzle ORM + better-sqlite3 persistence with a transactional migration runner, pluggable AuthContext, error envelope, structured logging, and a `health.ping` procedure. Establishes the patterns every subsequent plan (#2–#8) will mirror.

## User Story
As a developer building VibeMaestro,
I want a running Electron+Bun monorepo with persistent storage and a typed IPC contract,
So that subsequent plans can add Task/Run/Agent resources, the live event bus, and the xterm.js terminal without re-litigating cross-cutting concerns.

## Problem → Solution
- **Current:** repo contains only `DESIGN.md`, `API.md`, `README.md`, `design-tokens.json`, `design-preview.html`, `assets/logo.svg`. No source code, no tooling, no persistence.
- **Desired:** `bun install && bun dev` opens an Electron window. The renderer issues a typed `health.ping()` tRPC call over IPC and receives `{ status: "ok", version, uptime_ms }`. SQLite is open at `~/.vibemaestro/data.sqlite`, migrations have run, schema is empty (resources land in plan #2). `bun typecheck`, `bun lint`, `bun test` all pass.

### NAMING_CONVENTION (pattern this plan establishes)

```ts
// File names: kebab-case
//   apps/desktop/src/main/routers/health.ts
//   packages/db/src/migrate.ts
//
// TypeScript identifiers:
//   - Types/Interfaces: PascalCase                     type AuthContext = ...
//   - Classes: PascalCase                              class AppError extends Error
//   - Functions / methods / variables: camelCase       function createDb()
//   - Constants: SCREAMING_SNAKE_CASE                  const VIBEMAESTRO_HOME = ...
//   - tRPC procedure paths: dot.kebab in path,         router({ ping: t.procedure... })
//     camelCase in name                                trpc.health.ping.query()
//   - Error codes (over-the-wire): snake_case          "validation_error", "not_found"
//   - Database tables/columns: snake_case              CREATE TABLE tasks (created_at ...)
//   - JSON field names (over IPC envelope): snake_case "request_id", "exit_code"
//
// Imports: prefer named imports; group as
//   1) builtin / node:
//   2) third-party
//   3) workspace packages (@vibemaestro/*)
//   4) relative
// Biome enforces ordering automatically.
```

### ERROR_HANDLING (canonical AppError + tRPC error formatter)

```ts
// apps/desktop/src/main/errors.ts
export type ErrorCode =
  | "validation_error"
  | "not_found"
  | "invalid_state"
  | "conflict"
  | "agent_unavailable"
  | "internal_error";

export class AppError extends Error {
  constructor(
    public readonly code: ErrorCode,
    message: string,
    public readonly details?: unknown,
    public readonly statusHint?: number, // for v2 HTTP mirror; ignored over IPC
  ) {
    super(message);
    this.name = "AppError";
  }
}

// Map to the over-the-wire envelope { error: { code, message, details? } }.
// Used by tRPC's errorFormatter and (in v2) by Hono's onError.
export function toEnvelope(err: unknown, requestId: string) {
  if (err instanceof AppError) {
    return { error: { code: err.code, message: err.message, details: err.details, request_id: requestId } };
  }
  // Never leak internal details to the renderer. Log full stack server-side.
  return { error: { code: "internal_error", message: "Internal error", request_id: requestId } };
}
```

```ts
// apps/desktop/src/main/trpc.ts — tRPC initialization with envelope formatter
import { initTRPC } from "@trpc/server";
import { ZodError } from "zod";
import { AppError, toEnvelope } from "./errors";
import type { AuthContext } from "./middleware/auth";

export type Context = { auth: AuthContext; requestId: string; logger: import("pino").Logger };

const t = initTRPC.context<Context>().create({
  errorFormatter({ shape, error, ctx }) {
    const requestId = ctx?.requestId ?? "unknown";
    if (error.cause instanceof ZodError) {
      return {
        ...shape,
        data: {
          ...shape.data,
          envelope: {
            error: {
              code: "validation_error",
              message: "Request validation failed",
              details: error.cause.issues.map((i) => ({
                field: i.path.join("."),
                message: i.message,
                code: i.code,
              })),
              request_id: requestId,
            },
          },
        },
      };
    }
    if (error.cause instanceof AppError) {
      return { ...shape, data: { ...shape.data, envelope: toEnvelope(error.cause, requestId) } };
    }
    return { ...shape, data: { ...shape.data, envelope: toEnvelope(error, requestId) } };
  },
});

export const router = t.router;
export const procedure = t.procedure;
export const middleware = t.middleware;
```

### IPC_BRIDGE_PATTERN (custom tRPC link — Electron IPC, no third-party lib)

```ts
// apps/desktop/src/main/ipc.ts — the server side
import { ipcMain } from "electron";
import { callProcedure } from "@trpc/server";
import { ulid } from "ulid";
import { appRouter } from "./routers/_app";
import { logger, childLogger } from "./lib/logger";
import { authMiddleware } from "./middleware/auth";

export function registerIpcHandlers() {
  ipcMain.handle("trpc.invoke", async (_event, req: { id: string; path: string; type: "query" | "mutation"; input: unknown }) => {
    const requestId = ulid();
    const reqLogger = childLogger({ request_id: requestId, procedure: req.path, type: req.type });
    const start = Date.now();
    try {
      const auth = await authMiddleware();
      const result = await callProcedure({
        procedures: appRouter._def.procedures,
        path: req.path,
        rawInput: req.input,
        ctx: { auth, requestId, logger: reqLogger },
        type: req.type,
      });
      reqLogger.info({ duration_ms: Date.now() - start, ok: true });
      return { ok: true, data: result };
    } catch (err) {
      reqLogger.error({ duration_ms: Date.now() - start, err }, "procedure failed");
      // Tease out the envelope from tRPC's TRPCError shape (set by errorFormatter)
      const envelope = (err as any)?.shape?.data?.envelope ?? { error: { code: "internal_error", message: "Internal error", request_id: requestId } };
      return { ok: false, envelope };
    }
  });
}
```

```ts
// apps/desktop/src/renderer/trpc.ts — tRPC client with custom IPC link
import { createTRPCProxyClient, TRPCLink } from "@trpc/client";
import { observable } from "@trpc/server/observable";
import type { AppRouter } from "../main/routers/_app";

declare global {
  interface Window {
    vmBridge: {
      trpcInvoke: (req: { id: string; path: string; type: "query" | "mutation"; input: unknown }) =>
        Promise<{ ok: true; data: unknown } | { ok: false; envelope: { error: { code: string; message: string; details?: unknown; request_id: string } } }>;
    };
  }
}

const ipcLink: TRPCLink<AppRouter> = () => ({ op }) =>
  observable((observer) => {
    let cancelled = false;
    window.vmBridge.trpcInvoke({ id: op.id.toString(), path: op.path, type: op.type, input: op.input })
      .then((res) => {
        if (cancelled) return;
        if (res.ok) {
          observer.next({ result: { type: "data", data: res.data } });
          observer.complete();
        } else {
          observer.error({ message: res.envelope.error.message, data: res.envelope } as any);
        }
      })
      .catch((err) => !cancelled && observer.error(err));
    return () => { cancelled = true; };
  });

export const trpc = createTRPCProxyClient<AppRouter>({ links: [ipcLink] });
```

### DB_CLIENT_PATTERN (Drizzle + better-sqlite3 singleton)

```ts
// packages/db/src/client.ts
import Database from "better-sqlite3";
import { drizzle, BetterSQLite3Database } from "drizzle-orm/better-sqlite3";
import * as schema from "./schema";

export type Db = BetterSQLite3Database<typeof schema>;

let _db: Db | null = null;

export function createDb(filePath: string): Db {
  if (_db) return _db;
  const sqlite = new Database(filePath);
  sqlite.pragma("journal_mode = WAL");
  sqlite.pragma("foreign_keys = ON");
  sqlite.pragma("synchronous = NORMAL");
  _db = drizzle(sqlite, { schema });
  return _db;
}

export function closeDb() {
  if (_db) {
    (_db as any).$client.close();
    _db = null;
  }
}
```

### MIGRATION_PATTERN (transactional file-based migrations, idempotent)

```ts
// packages/db/src/migrate.ts
import { migrate as drizzleMigrate } from "drizzle-orm/better-sqlite3/migrator";
import { logger } from "../../../apps/desktop/src/main/lib/logger"; // or move logger to a shared package later
import type { Db } from "./client";

export function runMigrations(db: Db, migrationsFolder: string): void {
  const start = Date.now();
  drizzleMigrate(db, { migrationsFolder });
  logger.info({ duration_ms: Date.now() - start }, "migrations applied");
  // Drizzle internally creates a __drizzle_migrations table and is idempotent;
  // re-runs are safe and effectively no-op when up-to-date.
}
```

### TEST_STRUCTURE (Vitest-compatible via `bun test`; tRPC caller for unit tests)

```ts
// apps/desktop/test/server.test.ts
import { describe, it, expect, beforeAll } from "bun:test";
import { createCallerFactory } from "@trpc/server";
import { appRouter } from "../src/main/routers/_app";
import { authMiddleware } from "../src/main/middleware/auth";
import { logger } from "../src/main/lib/logger";

const createCaller = createCallerFactory(appRouter);

async function makeCaller() {
  const auth = await authMiddleware();
  return createCaller({ auth, requestId: "test-req", logger: logger.child({ test: true }) });
}

describe("health.ping", () => {
  it("returns ok with version and uptime", async () => {
    const caller = await makeCaller();
    const res = await caller.health.ping();
    expect(res.status).toBe("ok");
    expect(typeof res.uptime_ms).toBe("number");
    expect(res.version).toMatch(/^\d+\.\d+\.\d+/);
  });
});

describe("error envelope", () => {
  it("formats AppError with request_id", async () => {
    const caller = await makeCaller();
    await expect(caller.health.fail()).rejects.toMatchObject({
      data: { envelope: { error: { code: "internal_error", request_id: expect.any(String) } } },
    });
  });
});
```

## Testing Strategy

### Unit Tests

| Test | Input | Expected Output | Edge Case? |
|---|---|---|---|
| `health.ping` returns ok | none | `{ status: "ok", version: string, uptime_ms: number }` | no |
| `health.fail` returns envelope | `VIBEMAESTRO_TEST=1` | tRPC error with `data.envelope.error.code === "internal_error"` and `request_id` set | yes (test-only flag) |
| `health.fail` returns not_found | `VIBEMAESTRO_TEST` unset | tRPC error with `code === "not_found"` | yes |
| `errorFormatter(ZodError)` | a Zod parse failure | `{ error: { code: "validation_error", details: [...] } }` | yes |
| `errorFormatter(AppError)` | `new AppError("conflict", "...")` | `{ error: { code: "conflict", message: "..." } }` | yes |
| `errorFormatter(plain Error)` | `new Error("boom")` | `{ error: { code: "internal_error", message: "Internal error" } }` (no leak) | yes |
| `authMiddleware()` returns local context | none | `{ userId: "local", scopes: ["*"] }` | no |
| `vibemaestroHome()` creates dir | `HOME=<tempdir>` | dir exists at `<tempdir>/.vibemaestro` | yes (no pre-existing dir) |
| `getDb()` is idempotent | called twice | same instance returned | no |
| `runMigrations()` is idempotent | run twice on the same DB | second call is no-op (no error) | yes |

### Edge Cases Checklist

- [ ] First-launch: `~/.vibemaestro/` does not exist → created with mode 0700
- [ ] Second-launch: `data.sqlite` exists → migrations no-op, app boots
- [ ] DB file unwritable (read-only home) → `getDb()` throws AppError("internal_error") (logged, not silent)
- [ ] Concurrent IPC calls → request_ids are unique (verify ULID uniqueness across 1000 sequential generations)
- [ ] Renderer DevTools console is clear after window opens (no errors, no warnings)
- [ ] Bridge: input larger than 1 MB still serializes (synthetic test; sets a guardrail before plan #5's binary terminal frames)

## CI/CD (added by eng review 2026-05-09 — lives in PR #1a)

`.github/workflows/ci.yml` runs on every PR and on pushes to `main`:

```yaml
name: ci
on:
  pull_request:
  push:
    branches: [main]

jobs:
  test:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: 1.3.x
      - run: bun install --frozen-lockfile
      - run: bun typecheck
      - run: bun lint
      - run: bun test
```

**Why both Ubuntu and macOS:** the native modules (`better-sqlite3`, `node-pty`) rebuild against the platform's Node ABI. A passing macOS-only run won't catch a Linux ABI break. Windows is added in plan #9 (release pipeline) when packaging requires it.

**What this catches starting at PR #2:** any plan that breaks typecheck, lint, or unit tests fails the PR before merge. Without CI, plans #2-#8 land based on the implementer's local checks alone.

**Build-installer / E2E / publish workflows** are deferred to plan #9.

## Database portability discipline (added by eng review 2026-05-09 — lives in PR #1c)

v1 ships SQLite. The data layer is architected so a future postgres swap is a driver-and-init change, not a multi-week refactor.

### Five rules (enforced by code review and the CLAUDE.md "Database access" rule)

1. **Repository pattern.** All data access goes through `packages/db/src/repositories/*.ts`. Services consume `TaskRepository`, `RunRepository`, `AgentRepository` interfaces — never `import { drizzle }` or `import { sqliteTable }` outside `packages/db/`.
2. **Portable Drizzle idioms only in repositories.** Use `.onConflictDoNothing()` instead of raw `INSERT OR IGNORE`. Use Drizzle's `relations()` for joins. Use ULIDs (already specified in plan #2) and a sequence-table for the `VM-N` slug (already specified). No `INTEGER PRIMARY KEY AUTOINCREMENT`.
3. **SQLite-specific init isolated.** All `PRAGMA` statements live in `packages/db/src/dialects/sqlite-init.ts` and run on connection open. Postgres swap = replace this file with `postgres-init.ts` (or skip; pg defaults are fine).
4. **Migration directory split.** `packages/db/migrations/sqlite/*.sql` ships in v1. `packages/db/migrations/postgres/*.sql` is created when needed (drizzle-kit emits both). The migration runner picks based on `DATABASE_URL` scheme (`sqlite:///path` vs `postgres://…`).
5. **CHECK constraints documented as dialect-specific.** Plan #2's "edit migration to add CHECK" GOTCHA gets a follow-up note: when generating the postgres migration, regenerate from the Drizzle schema rather than porting the SQL by hand.

### WAL mode (sqlite-init.ts)

```ts
db.pragma("journal_mode = WAL");
db.pragma("synchronous = NORMAL");
db.pragma("wal_autocheckpoint = 1000");
db.pragma("foreign_keys = ON");
```

WAL gives crash recovery + concurrent reader-during-write (matters for plan #3's PTY byte-counter writing while plan #2's UI mutations run). Orthogonal to portability; postgres has its own durability defaults.

## Contract test (transport-agnostic surface — added by eng review 2026-05-09 — lives in PR #1c)

API.md §5-§7 are documented as transport-agnostic. v1 ships them over Electron IPC; v2 will mirror over HTTP/SSE/WebSocket. Without a contract test, the v2 mirror can silently drift from the v1 IPC contract.

**What ships in PR #1c:**
- `packages/core/src/contracts/` — Zod schemas for every resource (`Task`, `Run`, `Agent`), every event payload (`run.started`, `run.progress`, `run.ended`, `task.state_changed`, `agent.availability_changed`), and the error envelope. (Plans #2/#3/#4 already specify these as Zod; this rule says: keep them in `@vibemaestro/core/contracts`, don't inline them in router files.)
- `apps/desktop/test/contract.test.ts` — dumps the full tRPC router shape to JSON schema, snapshots to `apps/desktop/test/__snapshots__/contract.json`, fails if the snapshot drifts unintentionally.
- Routers (`tasks`, `runs`, `agents`) consume schemas from `@vibemaestro/core/contracts` via `.input(taskCreateSchema)` / `.output(taskSchema)`. Future v2 HTTP layer consumes the same schemas.

**Maintenance:** when a contract changes intentionally, the implementer runs `bun test --update-snapshots` and the diff IS the API change. Reviewers see the contract drift in the snapshot diff.
